SV2

Cross-Tab ▾
Opens data:,Hello in right split tab, then the iframe (in left tab) calls win.alert() on the right tab's window object. The dialog appears in the right tab but was initiated from the left tab's iframe. This is the cross-tab equivalent of the original bug — tests whether split view changes origin attribution.

Click Run — data:,Hello opens in RIGHT tab
2

3

SV4
Named target reuse across split tabs
Named Target ▾
Opens a named target tab (_splitTarget) twice. First open navigates it to data:,First. Second open reuses the same tab and navigates to a javascript: URL. Tests whether reusing a named tab changes NavigationEntry state and origin attribution.
Steps
1
Click Run — right tab opens as "data:,First"
2
After 800ms — same tab navigates to javascript:alert()
3
Check dialog title in right tab — second navigation origin
❌ Vulnerable

✅ Safe

// First open — named target tab created

  );

SV7
Right tab navigates cross-origin, then dialog fires
Cross-Origin Nav ▾
Opens right tab to data:,Hello, then navigates it to a real cross-origin page (https://example.com), waits for load, then tries to call win.alert() from the iframe. After a cross-origin navigation, Chrome should block the cross-origin win.alert() call entirely. Tests whether split view + cross-origin navigation creates any unexpected access.
Steps
1
Right tab opens as data:,Hello
2
After 500ms — navigates to example.com (now cross-origin to iframe)
3
After 2s — iframe tries win.alert() — should be blocked
4
Check activity log below for result
❌ Vulnerable
Dialog fires on example.com tab showing wrong origin — cross-origin access should not work
✅ Expected
Console error: cross-origin access blocked. No dialog fires.
// Right tab navigates cross-origin — iframe tries to dialog it
var win = window.open('data:,Hello', '_blank');
setTimeout(() => {
  win.location.href = 'https://example.com'; // cross-origin nav
  setTimeout(() => {
    try {
      win.alert('SV7: Should be blocked after cross-origin nav');
    } catch(e) {
      console.log('SV7 correctly blocked:', e.message);
    }
  }, 2000);
}, 500);
SV8
Blob URL opened in right split tab
Blob + Split ▾
Creates a Blob URL owned by the iframe's origin and opens it in the right split tab. The blob page calls alert() from within itself. Blob URLs are well-defined — their origin equals their creator's origin (this iframe). In split view, checks if the right tab correctly shows iframe origin in dialog.
❌ Vulnerable
Blob page dialog shows 1.hackeronce.com says — top frame origin leaked despite blob being from iframe
✅ Safe
Blob page dialog shows sho3hit.github.io says — correct creator origin
// Blob URL created by iframe, opened in right split tab
const html = `<script>
  alert('SV8: Blob in split right tab\\nBlob creator: sho3hit.github.io\\nwin.origin: ' + window.origin);
</script>`;
const blob = new Blob([html], { type: 'text/html' });
window.open(URL.createObjectURL(blob), '_blank');

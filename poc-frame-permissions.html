Permission Test Cases
📷
Camera Access
getUserMedia({video: true})
▾
Requests camera access from the cross-origin iframe. The permission prompt should show the iframe's domain (sho3hit.github.io). If it shows 1.hackeronce.com — the top frame origin is being spoofed in the permission dialog.
Steps
1
Click Run — permission prompt appears below address bar
2
Read the domain name in the prompt before clicking anything
3
Click Block (don't actually grant camera access)
4
Record result using buttons below
❌ Vulnerable
Prompt says:
1.hackeronce.com wants camera
✅ Safe
Prompt says:
sho3hit.github.io wants camera
// Triggers camera permission prompt from cross-origin iframe
navigator.mediaDevices.getUserMedia({ video: true })
  .then(s => { s.getTracks().forEach(t => t.stop()); log('P01: Granted','ok'); })
  .catch(e => log('P01: '+e.message,'warn'));

Notification Permission
Notification.requestPermission()
▾
Requests notification permission from within the cross-origin iframe. This is a high-value test — notification permission prompts are frequently targeted in phishing because they appear very prominently. If origin is spoofed here, attacker could trick user into granting notifications to the top frame while thinking it's the iframe.
Notification permission in iframes requires the allow="notifications" attribute on the iframe tag — already set in the updated parent-page.html.
❌ Vulnerable
Prompt shows:
1.hackeronce.com wants notifications
✅ Safe
Prompt shows:
sho3hit.github.io wants notifications
Notification.requestPermission()
  .then(r => log('P04: Notification result: '+r, r==='granted'?'ok':'warn'))
  .catch(e => log('P04: '+e.message,'err'));

Clipboard Write Permission
navigator.clipboard.writeText()
▾
Requests clipboard write access from the iframe. Unlike clipboard read, write access typically doesn't require a user prompt in Chrome (granted automatically with user gesture). Tests whether the permission API check itself attributes to correct origin.
Write may succeed silently without a prompt — check the activity log for result.
❌ Vulnerable
If prompt appears — shows 1.hackeronce.com
✅ Safe
Prompt shows sho3hit.github.io or write succeeds silently
navigator.clipboard.writeText('P07 test — origin: '+window.origin)
  .then(() => log('P07: Write succeeded (no prompt needed)','ok'))
  .catch(e => log('P07: '+e.message,'warn'));

Screen Capture Permission
navigator.mediaDevices.getDisplayMedia()
▾
Requests screen capture access. Chrome shows a screen picker dialog that includes the requesting origin. This is a very sensitive permission — wrong origin shown here could trick user into sharing their screen with an unintended domain. Requires allow="display-capture".
Screen capture picker opens a separate system dialog. The origin appears in the Chrome dialog header above the picker.
❌ Vulnerable
Screen picker dialog header shows 1.hackeronce.com
✅ Safe
Dialog header shows sho3hit.github.io
navigator.mediaDevices.getDisplayMedia({ video: true })
  .then(s => { s.getTracks().forEach(t => t.stop()); log('P08: Screen granted','ok'); })
  .catch(e => log('P08: '+e.message,'warn'));
🔍
Permissions API State Query
navigator.permissions.query()
▾
Queries the current permission state for multiple permission types using the Permissions API — no prompt shown. This checks what origin Chrome associates internally with each permission query from the cross-origin iframe. Result appears in the activity log — checks if query is attributed to iframe or top frame internally.
No visible prompt for this test — all results appear in the Activity Log below.
// Query permission states — no prompt, results in log
const perms = ['camera','microphone','geolocation','notifications','clipboard-read'];
for (const p of perms) {
  navigator.permissions.query({name: p})
    .then(r => log('P09 '+p+': '+r.state, r.state==='granted'?'ok':'info'))
    .catch(e => log('P09 '+p+' error: '+e.message,'warn'));
}
